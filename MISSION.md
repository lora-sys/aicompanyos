# Mission: AI Company OS 架构设计哲学

## 为什么我想学这个

我正在构建 AI Company OS — 一个用 AI Agent 模拟公司运作的系统。核心架构基于 **Loop Engineering（循环工程）** 范式，通过双层嵌套循环（Outer Loop / Inner Loop）驱动 Agent 协作完成复杂任务。

当前进展：
- 已实现内容产出部（Content Production Department），基于 ADR-005 部门制架构
- ADR-005 符合度 95/100，方向正确
- 下一步演进方向：
  1. **动态团队**：任务特征驱动的 Agent 组合（不是固定配对）
  2. **成长护城河**：Memory 回流闭环（沉淀物→复用→越用越好）
  3. **知识资产体系**：Prompt 知识库、成功案例库、用户画像

## 我想要掌握的核心能力

1. **系统思维**：理解为什么"部门=配置剖面"而不是"部门=独立系统"
2. **动态编排**：理解如何设计任务特征驱动的团队组合机制
3. **进化架构**：理解如何让系统通过使用不断变强（护城河）
4. **抽象分层**：理解 loop-engine / departments / team 各层的职责边界

## 学习目标

学完后我能：
- 解释 AI Company OS 的核心设计决策及其理由
- 判断一个新功能应该放在哪一层
- 设计新的 Department 和 Team 规则
- 评估架构方案的质量和风险
